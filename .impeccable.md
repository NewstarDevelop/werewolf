## Design Context

### Users

**主受众**：想玩狼人杀但不愿意等匹配、约人、开车开麦的 solo 玩家。
- **使用场景**：一人独处时（午休、睡前、通勤间隙），打开浏览器想打一局 10–20 分钟的推理局
- **人群画像**：18–35 岁、玩过线下或线上狼人杀、对"等人组局"疲惫、对 AI 陪玩有好奇心
- **他们想完成的事**：秒开局 → 沉浸进一局真正有推理与伪装张力的狼人杀 → 结局后带着"打得还行 / 被坑了 / AI 演技不错"的情绪离场
- **反模式受众**：不懂狼人杀规则的纯新手需要 onboarding 兜底（见下文 Principle 3）；硬核开黑玩家不是我们第一目标，功能能满足即可

**现实约束**：
- 单人无队友，所有互动对手都是 AI
- 纯文本交互、无语音
- 每局 9 人（1 真人 + 8 AI）、固定身份板（3 狼 / 3 民 / 3 神：预言家·女巫·猎人）

### Brand Personality

**三词调性：墨色 · 克制 · 松动**

- **墨色 (ink)** — 视觉基调是东方水墨 / 碳笔 / 活字印刷的含蓄色谱，不是西洋油画的饱和。主色是不同浓度的墨，不是彩虹
- **克制 (restraint)** — 装饰极少、留白极大、装饰性副文本全删。每一个元素都要能解释"为什么在这里"
- **松动 (ease)** — 不是死板的极简主义，允许纸张纤维、印章位移、手写不工整等"不完美的温度"。排版有呼吸，不要把一切对齐到像素网格

**目标情绪**：
- 进入时：**安静、专注**（像翻开一本线装书）
- 推理中：**克制的紧张**（不是 jump-scare 的紧张，是棋局式的压迫感）
- 关键决策前：**分量感**（destructive 动作必须有仪式，不是随手一点）
- 终局：**留白收尾**（帷幕感，而非烟花感）

**错觉与反情绪**：
- ❌ 不要"激情澎湃"、"打了鸡血"、"酷炫"
- ❌ 不要"可爱"、"萌"、"游戏化奖章"
- ❌ 不要 SaaS Dashboard 的"专业工具感"
- ❌ 不要赛博朋克 / 霓虹 / 全彩渐变

### Aesthetic Direction

**基底参考**：Notion / Obsidian 的极简文稿排版 × 东方水墨桌游剧本

- **不是**：瓷器釉色 / 卷轴金粉 / 山水画插图 / 国风游戏 UI 的华丽装饰
- **是**：一本活字印刷的狼人杀剧本在屏幕上展开。纸是宣纸 / 米纸的纹理，字是墨迹，座位是盖下去的印章，死亡是落下的墓碑石

**Light 主题（白昼 · 默认）**
- 底：宣纸白 `oklch(97% 0.01 90)` — 微泛米色，不是纯白
- 文：松烟墨 `oklch(22% 0.02 250)` — 墨色带一丝蓝底调，不是纯黑
- 辅：墨灰 `oklch(55% 0.02 250)` — 副文本、时间戳
- 强调：朱砂印 `oklch(52% 0.18 25)` — 只用在 destructive（狼刀、毒药、猎人开枪、死亡公告）和关键焦点。**每屏 ≤ 3 处**
- 次强调：琥珀金 `oklch(62% 0.1 75)` — 用于"你"的身份标、预言家查验结果等需要温度的高光
- 装饰：极少，仅作印章 / 纹样底纹 / 分隔线

**Dark 主题（夜晚 · 手动切换或阶段驱动）**
- 底：泼墨深蓝 `oklch(18% 0.04 250)` — 带墨香的深夜色，不是纯黑，也不是 AI 默认深紫
- 文：月白 `oklch(92% 0.015 85)` — 宣纸白的镜像，偏暖
- 辅：雾灰 `oklch(65% 0.015 250)`
- 强调：朱砂印在深底上稍调亮 `oklch(62% 0.2 25)`
- 次强调：灯笼金 `oklch(72% 0.13 75)`

**字体栈**
- 主标题：Source Han Serif SC / Noto Serif SC / Songti SC（保留现有选择）
- 正文：Source Han Sans SC / Noto Sans SC（保留）
- 特色备选：霞鹜文楷（LXGW WenKai）— 用于身份卡、终局、特殊播报
- 西文数字：可用 tabular-nums 确保座位号等数字对齐
- 绝不使用：Inter / Roboto / Arial / 系统默认无衬线作主标题

**布局哲学**
- **单焦点 + 文稿节奏**：任意时刻屏幕只有一个主角（多数时候是桌面/座位，轮到你时是操作面板）。其余元素后退
- **桌面为中心**：座位不是 3×3 方格 Dashboard，而是**围坐一桌**（环形 / 拱形 / 桌沿聚焦），让视觉有桌游感
- **文稿式聊天流**：对局日志不是聊天气泡，而是**剧本式印刷段落**，不同信息类型通过首行缩进、字号、墨色浓度区分，不用色块卡片
- **装饰元素仅 3 类**：印章（身份/关键标记）、墨渍（仪式动画）、纸纹（背景底纹）。其它装饰全删

**动效语言**
- **默认 `ease-out-quart` / `ease-out-expo`** — 像落笔、像纸落桌面
- **禁用 bounce / elastic / 弹簧**
- **日夜切换用"泼墨过渡"** — 从一点扩散出墨色覆盖全屏
- **死亡揭示仪式**：翻牌 → 墓碑落 → 印章变灰 → 整条序列 ≤ 2.4s
- **全动效遵守 `prefers-reduced-motion`**

### Design Principles

**原则 1：墨胜于色**
> 先用墨色层次（浓淡）建立视觉层级，再用朱砂/琥珀做 ≤ 10% 的强调。任何诱惑引入第三种强调色的时候，都应该改用"加重墨色或加大字重"替代。

**原则 2：留白即信息**
> 屏幕 40% 以上应当是留白。信息密度靠字号/字重/缩进/行距打，不靠卡片、边框、背景色来分区。从"每个容器都要有 background + border + radius"改为"多数容器只是一段文字"。

**原则 3：文人不喊叫**
> 所有 eyebrow caps、英文装饰大字、"Stage Of Intrigue / Seat Board / Narrative"等都要删。保留的 kicker 应该是中文、低调、仅作功能性分节（例如"白昼第二日"、"夜幕"）。代码常量（WOLF_KILL / REQUIRE_INPUT）永远不能泄漏到界面。

**原则 4：仪式是张力的来源**
> 狼人杀的张力来自关键时刻的仪式感而不是装饰堆砌。destructive 动作必须有二段确认；死亡揭示必须有仪式动画；日夜切换必须有过渡；终局必须有卷轴/帷幕/印章落款这类收束感。其余时刻应该极度安静。

**原则 5：新手能懂，老手不烦**
> 第一次轮到女巫时自动说"这局只能救一次 / 毒一次"；第二次不再说。角色技能说明以"折叠抽屉"形式挂在身份卡下，老手默认折叠。键盘快捷键（1–9 选座、Enter 提交、Esc 取消）对老手无感存在，对新手不构成学习负担。

## Scope Boundaries（来自 /critique Phase 3）

- **禁区**：无。所有区块、组件粒度、布局、DOM 结构、文案都可动
- **推进方式**：12 步全套，按依赖顺序（teach → normalize → distill → clarify → harden → onboard → arrange → typeset → animate → delight → optimize → polish）
- **保留**：中文字体基础选择（Source Han 系列）、墨绿+暖铜+米白的基因（会用 OKLCH 重新定义但调性延续）
- **协议**：后端 WebSocket 契约（SYSTEM_MSG / CHAT_UPDATE / REQUIRE_INPUT / GAME_OVER）不动；若前端需要结构化字段（如死亡玩家 seatId 列表），由前端容错处理，不强制后端改
